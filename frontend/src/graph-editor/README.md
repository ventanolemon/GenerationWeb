# graph-editor — веб-канвас графов (скелет)

Клиентская часть `docs/architecture/graph_editor_api_contract.md`. Скелет:
доказывает, что модель состояния держит рекурсию вложенных подграфов; отдельные
типы узлов, ортогональная трассировка и авто-раскладка наращиваются поверх.

## Модель состояния (ключевое решение)

Источник правды — сам проводной JSON (`GraphSpec.to_dict()`-совместимый),
одно иммутабельное значение в `useReducer`. Тело `repeat`/`map` — вложенный
GraphSpec в `params.body`; «открыть тело» — это путь (`GraphPath`) в данные,
а не копия подграфа в состоянии редактора. Любая правка внутри тела —
иммутабельное обновление корня по пути (`updateAtPath`, structural sharing).
Следствия: сохранение = отдать состояние как есть; правки тела видны снаружи
немедленно; undo/redo при желании — история одного значения. Библиотека
состояния (Zustand и т.п.) не добавила бы ничего, кроме зависимости, —
селектор один (`getSubgraph(spec, path)`), экран один.

## Карта модулей

| Модуль | Что делает |
|---|---|
| `types.ts` | зеркало проводного формата §1 и каталога §2 |
| `model.ts` | чистые функции: мутации, путь-адресация подграфов, динамические порты (TS-зеркало правил ядра), совместимость типов, финальный узел |
| `store.tsx` | useReducer + контекст: spec (корень), path (крошки), selection |
| `geometry.ts` | детерминированная геометрия портов (без измерения DOM), цвета типов, безье |
| `api.ts` | `/api/graph/catalog\|validate\|preview` |
| `Canvas.tsx` | pan/zoom, перенос узлов, протяжка проводов (зелёный/янтарный по conversions), удаление |
| `NodeItem.tsx` | data-driven узел из каталога, бейдж «ВЫХОД» |
| `Palette.tsx` | категории из каталога, поиск, запрет TASK-узлов в подграфе |
| `ParamInspector.tsx` | одна форма по `params_schema`; `subgraph` → «Открыть тело…» |
| `PreviewPanel.tsx` | прогоны /graph/preview через существующий `<BlockRenderer>` |
| `GraphEditorModal.tsx` | тулбар (имя, крошки, предпросмотр, сохранить), дебаунс-валидация |

## Запуск (dev)

```bash
# generator_service (FastAPI) на :8000 — там /graph/* и вся семантика
python -m uvicorn generator_service.main:app --port 8000   # Python 3.12+
# фронт; без ASP.NET web_layer добавьте VITE_API_DIRECT=1
VITE_API_DIRECT=1 npm run dev
```

Вход в редактор: «+ Создать» → «Граф» (или «Изменить» на разделе
`constracted=4`). Сохранение — существующий upsert партиций, граф уходит в
`generation_params` дословно.

## Ограничения скелета (сознательные)

- провода — простые безье; ортогональная трассировка и раскладка по слоям —
  отдельный трек;
- рекурсия доказана на одном уровне (`repeat`/`map` body); навигация по ветвям
  `case` и shift-register UI — следующий этап (модель уже держит: ветвь —
  тот же `SubgraphRef` с ключом `case_i`);
- динамические порты в TS покрывают узлы скелета (formula/text/template/
  var_dict/block_list/repeat/map); остальные откатываются к статическому
  шаблону каталога — как `safe_ports` на десктопе;
- несовместимые типы с конвертером подсвечиваются янтарным и называют узел
  («вставьте …»), автоввод конвертера по отпусканию — не реализован;
- `file`/`hidden` в инспекторе — заглушка (ресурсный API вне контракта).
